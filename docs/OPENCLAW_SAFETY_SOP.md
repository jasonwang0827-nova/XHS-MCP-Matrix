# OpenClaw XHS Safety SOP

这份 SOP 给 OpenClaw 判断哪些动作可以自动执行、哪些动作必须谨慎处理。

## 1. 动作风险分级

### 低风险

可以自动执行：

```text
读取账号列表
检查登录状态
搜索笔记
保存搜索结果
生成线索报告
生成关注候选
生成线索报表
读取发布记录
```

### 中风险

建议默认生成任务，不直接执行：

```text
点赞
收藏
评论回复
关注候选
```

### 高风险

OpenClaw 自动流程可以按用户预先设定执行，但必须保证参数明确：

```text
真实发布小红书内容
真实评论
真实回复评论
真实关注账号
```

当前系统真实发布使用：

```json
{
  "submit": true,
  "approved": true,
  "approval_required": false,
  "execute": true
}
```

## 2. 成人用品 / 私密护理业务

必须做内容安全检查。

避免：

```text
露骨词
直接性暗示
医疗功效承诺
平台敏感表达
未成年人相关表达
```

默认策略：

```text
评论、回复、点赞、收藏、关注、私信都不自动执行。当前只导出线索报表，由人工处理。
```

## 3. 留学 / 移民业务

避免：

```text
保证录取
保证获签
保证移民成功
夸大成功率
冒充持牌顾问
```

建议表达：

```text
可以帮你梳理条件
可以先做初步评估
具体要看个人背景和材料
```

## 4. 盲盒 / 潮玩 / 工厂货源业务

避免：

```text
绝对低价
保证正品但无凭证
夸大供货能力
诱导违规交易
```

建议表达：

```text
可以了解款式和合作方式
具体库存以实际为准
可先沟通需求
```

## 5. 评论和私信

当前私信不能自动检查。

评论默认：

```text
只生成回复建议，不自动发送。
```

如果用户未来要求自动回复，OpenClaw 必须先确认：

```text
账号
回复范围
禁用词
是否允许真实发送
```

## 6. 发布前检查

发布前必须确认：

```text
account_id 正确
视频在该账号自己的 video-inbox
同名文案存在
标题不为空
正文不为空
标签不为空
没有明显违规词
max_publish_count 通常为 1
```

## 7. OpenClaw 不应该做的事

```text
不要猜 account_id
不要把所有账号素材混在一起
不要绕过登录检查
不要在私信能力不存在时假装已检查私信
不要把评论建议直接当成已发送
不要高频发布、搜索、点赞、收藏
```
